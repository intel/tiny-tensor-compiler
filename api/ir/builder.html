Builder#

The IR builder classes provide an alternative to providing intermediate representation in textual form. For example,

 1using namespace tinytc::ir;
 2
 3auto pb = program_builder{};
 4try {
 5    pb.create("gemm", [&](function_builder &fb) {
 6        auto A = fb.argument(memref_type(scalar_type::f32, {16, 4, dynamic}), "A");
 7        auto B = fb.argument(memref_type(scalar_type::f32, {4, 8}), "B");
 8        auto C = fb.argument(memref_type(scalar_type::f32, {16, 8, dynamic}), "C");
 9        fb.work_group_size(16, 1);
10        fb.body([&](region_builder &bb) {
11            auto gid = bb.create_group_id();
12            auto a = bb.create_subview(A, {slice(0, ir::dynamic), slice(0, ir::dynamic), gid});
13            auto c = bb.create_subview(C, {slice(0, ir::dynamic), slice(0, ir::dynamic), gid});
14            bb.create_gemm(transpose::N, transpose::N, value(1.0, scalar_type::f32), a, B,
15                           value(0.0, scalar_type::f32), c);
16        });
17    });
18} catch (compilation_error const &e) {
19    std::cerr << e.loc() << ": " << e.what() << std::endl;
20}

produces the following IR:

1func @sample(%A0: memref<f32x16x4x?>,
2             %B0: memref<f32x4x8>,
3             %C0: memref<f32x16x8x?>) work_group_size(16,1) {
4  %0 = group_id
5  %1 = subview %A0[0:?,0:?,%0] : memref<f32x16x4x?>
6  %2 = subview %C0[0:?,0:?,%0] : memref<f32x16x8x?>
7  gemm.n.n 0x1p+0, %1, %B0, 0x0p+0, %2
8    : f32, memref<f32x16x4>, memref<f32x4x8>, f32, memref<f32x16x8>
9}

In line 5 and 10 of the builder sample, functors are provided that accept a function_builder& or region_builder&, respectively. All commands issued to the builder within the functor are encapsulated in a region, thus, the hierarchical structure of the builder code is very similar to that of the IR it produces.

While the use of builders is less concise than writing IR in textual form, code generation can become much simpler. For example, if the shape of “A” in line 6 depends on some run-time computation, one can simply put a function call returning a shape there, instead of having to search-and-replace in textual IR. Another example is that several implementations of the functions body exist. These can be swapped out by providing a different functor to fb.body while the function_builder stays unaltered.

struct to_scalar_type#

Returns the scalar type corresponding to C++ type T

Specializations exist for bool, (u)int8_t, (u)int16_t, (u)int32_t, (u)int64_t, float, double. The scalar_type is stored in the static constexpr member “value”.
