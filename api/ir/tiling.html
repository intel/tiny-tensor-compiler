Work-group size#

Instructions like gemm, axpby, ger, etc., are processed by a work-group collectively. Choosing the work-group size appropriately is critical for performance. If the work_group_size and subgroup_size attribute are omitted, heuristics are used.

These work-group size heuristics can also be called manually. For example, if a kernel takes a memref of dynamic size, then the memref shapes are unknown at compile-time, and the automatic application of the heuristics is only giving a general-purpose work-group size. Therefore using the heuristics API, outlined below, one can supply memref shape constraints, like a maximum or typical shape.
