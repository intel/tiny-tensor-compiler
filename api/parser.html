Parser#

Programs written in the tensor language are either parsed using a source manager or the parse function. Using the source manager is the recommended method as it enhanced error messages with code context.

For example, if the source manager is set up as following

#include <tinytc/tinytc.hpp>

auto srcman = tinytc::source_manager(&std::cerr);
auto p = srcman.parse_file("test/codegen/type_mismatch0.ir");

and the source file contains an error, then the error message written to cerr contains context:

func @kernel(%K0: memref<f32>) {
  %0 = load %K0[] : memref<f64>
            ~~~~~~~~~~~~~~~~~~~
test/codegen/type_mismatch0.ir:6.13-31: Type of SSA value does not match operand type

Source manager#

class source_manager#

Source manager.

The source manager can parse tensor programs from files, stdin, or memory. Source code is stored in the manager such that error messages can be enhanced with code context.
